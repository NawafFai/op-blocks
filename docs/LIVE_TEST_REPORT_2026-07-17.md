## ✅ البوابة 2 — تسجيل COM على مستوى المستخدم (نجاح تام، قابل للتراجع)

بدون أي صلاحيات إدارية وبدون لمس HKLM: أداة `tools/live-verification/RegisterUser.cs`
تعيد توجيه HKCR إلى `HKCU\Software\Classes` (redirection قياسي للتثبيت
الشخصي) ثم تشغّل RegAsm البرمجي مع `[ComRegisterFunction]` الخاصة بـ CAPE-OPEN:

```
REGISTER OK  OPBlocks.Desalination.dll
REGISTER OK  OPBlocks.Electro.dll
REGISTER OK  OPBlocks.Lithium.dll
REGISTER OK  OPBlocks.Energy.dll
```

التراجع: `RegisterUser.exe <dll> /u` — لا أثر نظامي دائم.

## ⛔ المعطَّل — عائقان بيئيان على مستوى الجهاز (ليسا في كودنا)

1. **إضافة المكوّنات/استيراد .inp عبر الأتمتة يرجع E_POINTER**
   (`Components...TYPE.Elements.Add` وكل صيغ `InitFromFile/2` و`Import`).
   النمط يطابق عطل "Aspen Properties Enterprise Database" الموثق في
   HANDOFF §10.3 (يوقف مسارات Edit تحديدًا بينما يعمل المحرّك — وقد ثبت هنا
   أن المحرّك يعمل). قاعدة البيانات منطقة المالك حصريًا — لم ألمسها.
2. **حفظ وثيقة تحوي بلوك CAPE-OPEN يُسقط خادم الأتمتة** (RPC_E_SERVERFAULT
   عند SaveAs، متكرر 3/3، بينما حفظ وثيقة بلا بلوك CAPE-OPEN يعمل).
   المشتبه به الأول تداخل حماية Windows (Smart App Control المذكور في
   HANDOFF §3e مع DLLs المبنية حديثًا) أو نفس عطل قاعدة البيانات.
   العملية نفسها اشتغلت على الجهاز الآخر (أدلة §3/3c/3e) — فالمشكلة بيئة
   هذا الجهاز، لا الكود.

**النتيجة:** التشغيل المتقارب داخل اسبن لم يُنفَّذ اليوم على هذا الجهاز،
ولا يُدّعى. أدلة التقارب الحية القائمة تبقى أدلة الجهاز الآخر
(OP-RO: التوافق للرقم 686.484746 على IDEAL و28.24 بار على ELECNRTL).

## 🔬 كيف يكمل المالك آخر خطوة (10 دقائق، واجهة اسبن)

البلوكات مسجلة الآن لهذا المستخدم — ستظهر في اسبن مباشرة:
1. افتح Aspen Plus V14 (واجهة عادية).
2. استورد اللوحة `installer/aspen/ONE PROCESS.apm` أو أضف
   Model Library → CAPE-OPEN → ستجد OP-UF/OP-MED/OP-EDI... إلخ.
3. لكل بلوك تفحصه: أسقطه، تأكد من المنافذ، افتح الفورم (تبويبا
   Input/Results)، شغّل حالة، طابق النتائج مع التيّارات
   (خطوات §QC-CHECKLIST في HANDOFF كاملة).
4. إن ظهر خطأ قاعدة البيانات عند إضافة المكوّنات في الواجهة أيضًا،
   فالعطل نفسه الذي أوقف الأتمتة — وإصلاحه عندك (المنطقة المحظورة عليّ).

**الخلاصة القاطعة:** نظام Aspen Properties (AprSystem/mmg) على هذا الجهاز
معطوب لمسارات التحرير: يرفض إضافة المكوّنات (E_POINTER) وينهار عند حفظ أي

من OP-Blocks في كل الأدلة الجنائية. الجهاز الآخر (AprSystem سليم) شغّل كل

**فحص الحسم للمالك (دقيقة واحدة):** افتح واجهة اسبن → محاكاة جديدة فارغة →
